# 专业综合实训管理系统开发任务（按任务书对齐）

## 1. 总目标
- 严格覆盖任务书三端能力：学生端、指导教师端、系统管理员端。
- 完成“项目发布 -> 团队组建 -> 过程文档 -> 批阅反馈 -> 成绩评定 -> 统计分析”的闭环。
- 交付可运行原型与可答辩文档支撑代码。

## 2. 任务拆解（按优先级）

### A. 学生端（P0）
- A1 个人信息管理（查看、编辑基础信息）。
- A2 实训任务查看（按项目开设和时间节点查看任务）。
- A3 团队创建/加入/管理（成员维护、团队状态可见）。
- A4 开题/中期/结题过程文档提交与修改（版本记录）。
- A5 团队协作沟通（在线问答与回复）。
- A6 个人成绩查询（查看总评及评分构成）。

### B. 指导教师端（P0/P1）
- B1 班级与学生管理查询（按班级查看学生）。
- B2 项目创建与发布（任务书、考核标准、时间节点）。
- B3 学生团队审核（审核通过/驳回与状态说明）。
- B4 项目进度跟踪（任务完成率、提交统计）。
- B5 过程文档批阅与反馈（评分与文字反馈）。
- B6 在线答疑（问答主题、回复、关闭）。
- B7 成绩评定与导出（CSV/Excel 导出）。
- B8 教师负责项目统计（数量、提交、评分进展）。

### C. 系统管理员端（P0/P1）
- C1 Excel 模板批量导入用户信息（学生/教师）。
- C2 用户信息增删改查（含重置密码、状态管理）。
- C3 角色与权限配置（用户-角色绑定、权限点管理）。
- C4 学院/专业/班级基础数据维护。
- C5 系统运行状态与登录日志监控。
- C6 全院实训项目统计与分析。

### D. 测试与优化（P1）
- D1 功能测试（主流程、边界值、异常流程）。
- D2 性能测试（并发登录、列表查询、提交写入）。
- D3 安全测试（鉴权越权、SQL 注入、上传安全）。
- D4 问题闭环优化与回归验证。

## 3. 本轮补齐范围（当前迭代）
- 管理端缺口补齐：Excel 导入、角色权限基础、学院/专业/班级维护、登录日志。
- 教师端缺口补齐：在线答疑、团队审核、成绩导出、进度统计接口。
- 学生端缺口补齐：个人信息接口、过程文档阶段字段、问答入口。

## 4. 交付清单
- `backend/`：新增缺口模块 API 与服务实现。
- `frontend/`：新增基础数据与问答页面，补充导入/导出操作入口。
- `sql/schema.sql`：补充新表结构。
- `sql/seed.sql`：补充演示数据。
- `README.md`：新增模块使用说明。
